# Done Yet? UI QA

Decision: `PASS`

Tested locally on macOS in Chromium at 1440x1000 and 390x844.

## Behavior checked

- all six scenarios select and display the matching verdict, state, and checks;
- `All (11)` shows the complete contract and `Failed only` shows only non-passing checks;
- full contract and result JSON dialogs open and close;
- result JSON copies to the clipboard;
- export downloads the selected scenario report;
- generated UI data comes from the same verifier and fixtures as the CLI;
- desktop and mobile have no document-level horizontal overflow;
- controls remain keyboard-focusable with visible focus treatment;
- reduced-motion preferences disable non-essential animation and transitions;
- browser console contains no application error.

## Responsive note

The mobile scenario tray is horizontally scrollable by design. The selected scenario has a stable width, while the verification canvas, contract, evidence, and progress flow stack vertically. A narrow-layout bug that wrapped scenario names one letter per line was corrected before final capture.
